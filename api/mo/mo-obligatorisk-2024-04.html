Tidsplan

Fra 1. april 2024 vil det bli gjort endringer i hvilke dataelementer som er obligatoriske i innsending av informasjon for oppfylling av melde- og opplysningsplikten i Digitoll.
Endringene blir tilgjengelige for test i playground ca 1 måned før utrulling i produksjon.

Ingen av endringene innfører nye dataelementer i grensesnittene.
Endringen innebærer at dataelementer som hittil har vært frivillige blir obligatoriske.

Transportmelding for melde og opplysning alle transportmåter

Transportmelding er endret slik at planlagt ankomsttid sammen med estimert ankomsttid blir obligatorisk ved første innsending (POST).

Dataelementene som blir obligatoriske er:

estimatedDateAndTimeOfArrival
Dette er estimert ankomsttid. Ved første innsending (POST) skal denne og scheduledDateAndTimeOfArrival være identiske.
scheduledDateAndTimeOfArrival
Dette er planlagt ankomsttid. Ved første innsending (POST) skal denne og estimatedDateAndTimeOfArrival være identiske.

Forskjellen mellom planlagt ankomsttid og estimert ankomsttid er:

  • planlagt ankomsttid er den opprinnelig planlagte ankomsttiden for transportmiddelet til tollområdet, og kan aldri endres
  • estimert ankomsttid er den til en hver tid aktuelle estimerte ankomsttiden for transportmiddelet til tollområdet, og skal oppdateres ved endringer

For å unngå stans ved grensepasseringen anbefales det at innsender av transportmeldingen bruker oppdatering (PUT) til å oppdatere estimert ankomsttid så snart man er kjent med endringer i denne.

Transportmelding for melde og opplysning fly

Transportmelding for fly har ikke obligatorisk innsending av informasjon om fører. Dette blir obligatorisk, på samme måte som for transporter på vei og jernbane.
Dataelementene som blir obligatoriske er:

  • ActiveBorderTransportMeans → Operator
  • ActiveBorderTransportMeans → Operator → name
  • ActiveBorderTransportMeans → Operator → communication

Dataelementet communication skal fylles ute med type (telefon) og identifikator (telefonnummer).

Melding om hovedforsendelse (Master Consignment) for alle transportmåter

  • Informasjon om avsender og mottaker er frivillig på hovedforsendelse. Dette blir obligatorisk.
  • Det skal oppgis avsender og mottaker for hovedforsendelse
  • Det skal oppgis adresse for avsender og mottaker, der land, poststed, postnummer, og enten gateadresse eller postboksadresse er obligatorisk.
  • Dataelementene som blir obligatoriske er:
    • consignmentMasterLevel → consignee
    • consignmentMasterLevel → consignee → name
    • consignmentMasterLevel → consignee → typeOfPerson
    • consignmentMasterLevel → consignee → address
    • consignmentMasterLevel → consignee → address → country
    • consignmentMasterLevel → consignee → address → city
    • consignmentMasterLevel → consignee → address → postcode
    • en av: consignmentMasterLevel → consignee → address → streetLine
      eller consignmentMasterLevel → consignee → address → poBox
    • consignmentMasterLevel → consignor
    • consignmentMasterLevel → consignor → name
    • consignmentMasterLevel → consignor → typeOfPerson
    • consignmentMasterLevel → consignor → address
    • consignmentMasterLevel → consignor → address → country
    • consignmentMasterLevel → consignor → address → city
    • consignmentMasterLevel → consignor → address → postcode
    • en av: consignmentMasterLevel → consignor → address → streetLine
      eller consignmentMasterLevel → consignor → address → poBox

Melding om forsendelse (House Consignment) for alle transportmåter

  • Adresse for avsender og mottaker er frivillig på forsendelse. Dette blir obligatorisk.
  • Det skal oppgis adresse for avsender og mottaker, der land, poststed, postnummer, og enten gateadresse eller postboksadresse er obligatorisk
  • Dataelementene som blir obligatoriske er:
    • consignmentHouseLevel → consignee → address
    • consignmentHouseLevel → consignee → address → country
    • consignmentHouseLevel → consignee → address → city
    • consignmentHouseLevel → consignee → address → postcode
    • en av: consignmentHouseLevel → consignee → address → streetLine
       eller consignmentHouseLevel → consignee → address → poBox
    • consignmentHouseLevel → consignor → address
    • consignmentHouseLevel → consignor → address → country
    • consignmentHouseLevel → consignor → address → city
    • consignmentHouseLevel → consignor → address → postcode
    • en av: consignmentHouseLevel → consignor → address → streetLine
