Schedule

On April 1st 2024 changes will be made to the mandatory data elements for fulfilling obligation to notify and provide information upon arrival at the destination in the customs territory. These changes will be available in the playground for testing, approximately 1 month before production.

Which data elements are present in the interfaces will not change (no API structure changes). Some data elements that previously were not mandatory will now become mandatory.

Message for Transport (all modes of transport)

The message for transport is changed such that the planned (scheduled) time of arrival and the estimated time of arrival are now mandatory when sending in the message (POST).

The data elements that are now mandatory:

estimatedDateAndTimeOfArrival
This is the estimated time of arrival. At first submit (POST), this element and scheduledDateAndTimeOfArrival must be identical.
scheduledDateAndTimeOfArrival
This is the planned time of arrival. At first submit (POST), this data element and estimatedDateAndTimeOfArrival must be identical.

The difference between scheduled and estimated time of arrival is:

  • Scheduled time of arrival is the originally scheduled time of arrival for the transport to the customs area. This value can not be changed, and must be identical for each update.
  • Estimated time of arrival is the actual estimate, at any given time, for the arrival of the transport to the customs area. This value should be updated if changes to the estimated time of arrival become known.

To avoid unnecessary stops at the border, it is recommended that the submitter of the message for transport, utilities the update (PUT) to update the estimated time of arrival, as soon as any changes are known.

Message for Transport for mode of transport air

Message for Transport for air currently does not require submitting operator information. Operator information will now become mandatory information. This is now identical to road and train. The data elements that are now mandatory are:

  • ActiveBorderTransportMeans → Operator
  • ActiveBorderTransportMeans → Operator → name
  • ActiveBorderTransportMeans → Operator → communication

The data element communication should be submitted with type (telephone) and identification(telephone number).

Message for Master Consignment (all modes of transport)

  • Submitting information on consignor and consignee for the master consignment is currently not mandatory. This will now become mandatory.
  • It is mandatory to submit consignor and consignee for the master consignment.
  • It is mandatory to submit address for consignor and consignee for the master consignment.
  • The data elements that are now mandatory are:
    • consignmentMasterLevel → consignee
    • consignmentMasterLevel → consignee → name
    • consignmentMasterLevel → consignee → typeOfPerson
    • consignmentMasterLevel → consignee → address
    • consignmentMasterLevel → consignee → address → country
    • consignmentMasterLevel → consignee → address → city
    • consignmentMasterLevel → consignee → address → postcode
    • one of: consignmentMasterLevel → consignee → address → streetLine
      or consignmentMasterLevel → consignee → address → poBox
    • consignmentMasterLevel → consignor
    • consignmentMasterLevel → consignor → name
    • consignmentMasterLevel → consignor → typeOfPerson
    • consignmentMasterLevel → consignor → address
    • consignmentMasterLevel → consignor → address → country
    • consignmentMasterLevel → consignor → address → city
    • consignmentMasterLevel → consignor → address → postcode
    • one of: consignmentMasterLevel → consignor → address → streetLine
      or consignmentMasterLevel → consignor → address → poBox

Message for House Consignment (all modes of transport)

  • Address information on consignor and consignee for the house consignment is currently not mandatory. This will now become mandatory.
  • It is mandatory to submit address for consignor and consignee for the house consignment. Address for consignor and consignee for the house consignment must include country, city, postal code, and either street address or post office box number.
  • The data elements that are now mandatory are:
    • consignmentHouseLevel → consignee → address
    • consignmentHouseLevel → consignee → address → country
    • consignmentHouseLevel → consignee → address → city
    • consignmentHouseLevel → consignee → address → postcode
    • one of: consignmentHouseLevel → consignee → address → streetLine
      or consignmentHouseLevel → consignee → address → poBox
    • consignmentHouseLevel → consignor → address
    • consignmentHouseLevel → consignor → address → country
    • consignmentHouseLevel → consignor → address → city
    • consignmentHouseLevel → consignor → address → postcode
    • one of: consignmentHouseLevel → consignor → address → streetLine
      or consignmentHouseLevel → consignor → address → poBox
